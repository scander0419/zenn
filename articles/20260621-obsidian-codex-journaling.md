## はじめに

Obsidian でジャーナリングを続けようとして何度も失敗した理由の1つが、**書く時点で整理しようとして止まる**ことでした。

心理学や expressive writing の話を読んでいても、自分にとって大事だったのは「最初はとにかくダンピングしたい」という感覚です。そこで今回は、**書く時は完全に自由なダンピング、構造化は後から Codex Automations にやらせる**構成を作りました。

しかも、ただ日次を要約するだけではなく、

- 今日の未分析ノート
- 取りこぼしている直前の未分析ノート
- 過去の手帳ログ

も使って、`事実 / 解釈 / 認知の偏り / 次の一手` まで整理します。

再現用の最小セットは GitHub に公開しました。

https://github.com/scander0419/obsidian-codex-journaling-kit

## この記事でわかること

- `Obsidian` で自由にダンピングし、`Codex App Automations` で後から分析する構成
- なぜ「書く時は自由、分析は後から」の分離がジャーナリングと相性がいいのか
- 日次レビューと週次レビューをどう分けると反すうが増えにくいか
- そのまま試せる公開セットの中身

## 結論

先に結論を書くと、この構成で一番効いたのは次の分離でした。

- **人間はダンピングだけやる**
- **Codex は構造化だけやる**

ジャーナリングを続けにくくするのは、「書きながら自分を分析しようとすること」だと感じています。書いている最中に「これは事実か解釈か」「次の行動は何か」まで考え始めると、吐き出しが止まる。

なので、本文は散らかっていてよい前提にして、後から Codex に

- 事実
- 解釈
- 感情
- 認知の偏り
- 別の見方
- 次の一手

を分けさせるほうが、自分には圧倒的に続けやすかったです。

## 前提

自分の環境は次の通りです。

- 日付: 2026-06-21 時点
- OS: Windows 11
- ノート: Obsidian
- 自動分析: Codex App の Automations
- 主な保存先:
  - `01_ジャーナリング`: Obsidian の日次ノート
  - `00_手帳の`: 以前に手書きしていた手帳の文字起こし
  - `02_分析`: 週次レビューの出力先

Codex App の公式 docs では、Automations は `project`、`prompt`、`cadence`、`execution environment` を選んでスケジュールでき、project-scoped automation はローカルの Codex app が起動していて対象プロジェクトがディスク上にある必要があります。この前提で組んでいます。

## 作ったもの

公開したセットは、次の4層です。

### 1. ダンピングを書く場所

- `01_ジャーナリング`

ここに `YYYY-MM-DD.md` 形式で日次ノートを置きます。本文は本当に自由で、箇条書きでも散文でも OK です。

例えばこんな感じです。

```md
# 2026-06-21

今日は会議のあとをかなり引きずった。
責められたわけじゃないのかもしれないけど、
あの返し方で一気にしんどくなった。

## Codex分析
```

大事なのは、**本文をきれいにしようとしないこと**です。

### 2. 過去ログ置き場

- `00_手帳の`

ここには、以前に手書きしていた手帳の文字起こしを入れます。日次分析では毎回読むのではなく、**似たテーマが続いた時だけ補助参照**に使います。

### 3. 日次分析プロンプト

- `01_自動分析/codex_app_daily_automation_prompt_final.md`

このプロンプトでは、Codex に次をさせます。

- `今日` の未分析ノートを探す
- さらに `今日より前` のノートから、もっとも新しい `未分析ノート` を探す
- 両方あれば両方分析する
- `## Codex分析` に、`事実 / 解釈 / 認知の偏り / 次の一手` を追記する
- 必要な場合だけ `00_手帳の` を 1〜2 件だけ参照する

この「直前の未分析ノートも拾う」ロジックが地味に効きました。毎日きっちり書けない前提で、取りこぼしをそのまま残さないためです。

### 4. 週次分析プロンプト

- `01_自動分析/codex_app_weekly_review_prompt.md`

週次は `直近7日` を主対象にします。ただし比較のために、`01_ジャーナリング` と `00_手帳の` の全履歴を参照してよい設計にしました。

ここでのポイントは、**全部を均等に要約しないこと**です。見るのは主に次の2つです。

- 最近の変化
- 昔から続く長期傾向

## なぜこの設計にしたか

### ダンピングと分析を同じフェーズにしない

ジャーナリングの目的は、人によってかなり違いますが、自分にとってはまず

- 頭の中の負荷を外に出す
- 感情を詰まらせない
- 後からパターンを見る

の順でした。

この順番を崩して、最初から分析テンプレートに沿わせると、書くこと自体が重くなります。

なので今回の構成では、書いている時は分析を要求しません。分析は完全に `Codex` 側の仕事に寄せます。これは、expressive writing の「まず書く」部分と、認知再評価や problem solving の「後で整理する」部分を分けるイメージです。

### 週次は「全履歴を読む」より「今週を歴史の中に置く」

最初は週次レビューも「直近7日だけ」で十分かと思いました。ただ、それだと今週の波しか見えません。

一方で、全履歴を全部なめて総まとめしようとすると、今度は情報が重くなりすぎて、読んでも動けないレビューになりがちです。

そこで、週次は次の方針にしました。

- 主対象は直近7日
- 比較対象として全履歴を参照してよい
- 目的は網羅ではなく、`最近の変化` と `長期反復` を分けること

これだと、

- 今週だけ会議不安が増えているのか
- そもそも昔から `評価不安 -> 読心 -> 行動停止` の流れがあるのか
- 以前より相談行動は増えているのか

といった見方ができます。

## 実際の設定

公開セットには `AUTOMATION-SETUP.md` を入れましたが、ざっくり書くと設定は次の通りです。

### Obsidian

- Vault: ジャーナリングのルート全体
- Daily Notes の保存先: `01_ジャーナリング`
- テンプレート: なしでもよい

### Codex App の日次 Automation

- Project: ジャーナリングのルート
- Schedule: Daily
- Execution: local project
- Prompt: `codex_app_daily_automation_prompt_final.md` の全文

### Codex App の週次 Automation

- Project: ジャーナリングのルート
- Schedule: Weekly
- Execution: local project
- Prompt: `codex_app_weekly_review_prompt.md` の全文
- 出力先: `02_分析/weekly-review-YYYY-MM-DD.md`

## Obsidian Graph で見えるもの

この構成は、自動分析だけでなく Graph も面白いです。

タグとリンクのルールはかなり絞っています。

- 人物: `[[人名]]`
- 案件: `[[案件名]]`
- タグ: `#評価不安` `#読心` `#確認不足` など

ポイントは、感情タグを増やしすぎず、**分析ラベルを固定する**ことです。そうすると Graph や Local Graph で

- 誰に反応しやすいか
- どの案件で不安が出やすいか
- どの認知パターンが反復しているか

がかなり見やすくなります。

## ハマりどころ

### `read-only` だと動かない

日次分析はノートに追記するので、Codex App 側が `read-only` 相当だと失敗します。追記できる権限が必要です。

### 過去ログを読みすぎると重くなる

`00_手帳の` を常に広く読ませると、分析が長文化しやすくなります。補助参照は 1〜2 件に抑えるのがちょうど良かったです。

### 「気持ちに寄り添う分析」は意外と使いにくい

最初は優しいトーンの分析も試しましたが、毎日読むものとしては、

- 事実
- 解釈
- 偏り
- 次の一手

だけ返ってくるほうが、はるかに使いやすかったです。毎日読むものは、あまり情緒を足さないほうが回ります。

## まとめ

- ジャーナリングを続けるうえで、自分には **書く時の自由さ** が最優先だった
- なので `ダンピング` と `分析` を分離し、構造化は全部 Codex に後置きした
- 日次は `今日` と `直前の未分析ノート` を拾う
- 週次は `直近7日` を主対象にしつつ、全履歴を比較対象にして `最近の変化` と `長期傾向` を分ける
- Obsidian Graph では、感情そのものより `分析ラベル` を固定したほうが効く

「テンプレに沿うジャーナリングは続かないけど、ただ吐き出すだけでは終わらせたくない」という人には、かなり相性のいい構成だと思います。
